Class CallTreeController

Controller managing the Calls view.

Bound to Angular and to #/callTree

Warning:

The render loop should be as light and as fast as possible:

  • immediatly deffer with setTimeout when possible
  • use while loops whith length--

  • or for with cached length decreasing

  • Tasks that doesn't need to be computed every time should be deffered with a %

  • don't use switches

  • don't use try catch
  • don't use 'new'
  • don't acces the prototype chain
  • limit 'if' as much as possible

  • limit loops as much as possible

  • Most important : do not use console to print to stdout (console.log|warn|err). You will slow drastically the rendering and you may freeze the browser.

private raycaster

raycaster: any
  • Defined in callTreeController.ts:456

A THREEJS raycaster object.

Used to detect intersactions between mouse and an object.

explanation

When you want to detect a click in a 2D world, you check to mouse coordinates and your objects sizes and coordinate. If the mouse pointer is in the object, you consider the mouse is over it. In a 3D world you try to select an object with a 2D mouse (you screen is in 2D). To detect intersection between your mouse and an object, you cast a ray between your mouse pointer and the infinity. This ray is perpendicular to a plan facing your camera. If this ray touch an object. the mouse is over it.

private root3D

root3D: any
  • Defined in callTreeController.ts:470

ALL the scene is wrapped in a THREEJS.Object3D.

explanation

the camera is centrerd to look at (0,0,0). But our nodes are placed by D3js. D3 use (0,0,0) at the "top left corner". Our camera is centered with the scene but not with the objects. the simpler way (and the best for performances) is to put all the objects in an invisible, shapeless Object3D and to move this object to (-width/2, -heigth/2, -depth/2). By this way, our scene is centered
